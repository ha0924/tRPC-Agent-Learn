# 2.2 跑通 Python Quickstart

> 基于 `LlmAgent`、`Runner` 和模型配置，完成一个支持多轮对话与流式输出的基础 Agent

---

## 一、安装与配置

### 1. 安装 SDK

```bash
pip install trpc-agent-py python-dotenv
```

### 2. 配置环境变量

在 `quickstart_python/.env` 中设置：

```bash
TRPC_AGENT_API_KEY=your-api-key
TRPC_AGENT_BASE_URL=https://api.openai.com/v1   # 或混元/DeepSeek 的地址
TRPC_AGENT_MODEL_NAME=gpt-4o-mini               # 对应的模型名
```

框架通过 `OpenAIModel` 封装，支持所有 OpenAI 兼容接口（混元、DeepSeek、Moonshot 等）。

## 三、逐模块剖析

### 3.1 config.py — 模型配置

```python
def get_model_config() -> tuple[str, str, str]:
    api_key = os.getenv("TRPC_AGENT_API_KEY", "")
    base_url = os.getenv("TRPC_AGENT_BASE_URL", "")
    model_name = os.getenv("TRPC_AGENT_MODEL_NAME", "")
    # ... 校验
    return api_key, base_url, model_name
```

**要点**：
- 敏感信息通过环境变量管理，不硬编码
- 使用 `python-dotenv` 从 `.env` 文件加载

### 3.2 tools.py — 工具定义

三个工具演示了 Function Calling 的不同场景：

| 工具 | 参数 | 场景 |
|------|------|------|
| `get_weather(city)` | 城市名 | 外部 API 模拟 |
| `calculate(expression)` | 数学表达式 | 安全计算（白名单字符 + 沙箱 eval） |
| `get_current_time()` | 无参数 | 系统信息获取 |

**关键设计**：
- `calculate` 函数通过白名单字符过滤 + `eval` 隔离（`{"__builtins__": {}}`）防止代码注入
- 工具函数的 **docstring** 会被框架提取为 Function Schema 的 `description`
- 参数的 **类型标注** 会被提取为 JSON Schema 的 `type`

### 3.3 agent.py — Agent 声明

```python
root_agent = LlmAgent(
    name="assistant",
    description="支持天气查询、数学计算、时间查询的多功能助手",
    model=_create_model(),
    instruction=INSTRUCTION,
    tools=[
        FunctionTool(get_weather),
        FunctionTool(calculate),
        FunctionTool(get_current_time),
    ],
)
```

**核心概念映射**：
- `LlmAgent` — 框架的核心 Agent 类型，封装模型 + 工具 + 指令
- `OpenAIModel` — 模型适配层，统一 OpenAI 兼容接口的调用
- `FunctionTool` — 将普通 Python 函数封装为 Agent 可调用的工具
- `instruction` — System Prompt，定义 Agent 的角色和行为

### 3.4 main.py — Runner 与多轮对话

```python
# 创建 Runner
runner = Runner(app_name=app_name, agent=root_agent, session_service=session_service)

# 流式执行
async for event in runner.run_async(
    user_id=user_id,
    session_id=session_id,
    new_message=user_content,
):
    if event.partial:
        # 流式文本 chunk
        print(part.text, end="", flush=True)
    elif part.function_call:
        # 工具调用
    elif part.function_response:
        # 工具结果
```

**核心概念映射**：
- `Runner` — 运行器，驱动 Agent 执行循环
- `InMemorySessionService` — 内存会话管理，维护消息历史
- `run_async()` — 异步生成器，产出流式事件
- `session_id` 保持不变 → 实现多轮对话（历史消息自动累积）

## 四、事件流模型

`runner.run_async()` 返回的事件流包含以下类型：

```
┌──────────────────────────────────────────────────┐
│  用户输入                                         │
│     ↓                                            │
│  Runner 调度 Agent                                │
│     ↓                                            │
│  ┌────────────────────────────────────────────┐  │
│  │ Event(partial=True, text="北京")  ← 流式chunk │
│  │ Event(partial=True, text="今天")             │  │
│  │ Event(partial=True, text="天气晴朗")          │  │
│  └────────────────────────────────────────────┘  │
│     或                                           │
│  ┌────────────────────────────────────────────┐  │
│  │ Event(function_call=get_weather)  ← 工具调用  │
│  │ Event(function_response={...})    ← 工具结果  │
│  │ Event(partial=True, text=...)     ← 基于工具  │
│  │                                  结果的回复    │
│  └────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

| 事件属性 | 含义 |
|---------|------|
| `event.partial = True` | 流式文本 chunk，逐步输出 |
| `part.text` | 文本内容 |
| `part.function_call` | 模型决定调用的工具（名称 + 参数） |
| `part.function_response` | 工具执行结果 |
| `part.thought` | 模型思考过程（通常跳过） |

## 六、完整调用链路追踪

一次包含工具调用的对话请求，内部执行流程：

```
1. 用户输入 "北京天气怎么样"
2. main.py 将输入封装为 Content(parts=[Part.from_text(...)])
3. Runner.run_async() 被调用
4. Runner 从 SessionService 加载该 session_id 的历史消息
5. Runner 将历史 + 新消息 + System Instruction 拼接为 Prompt
6. Runner 调用 OpenAIModel.generate()，携带 tools schema
7. 模型返回 function_call: get_weather(city="北京")
   → Runner 产出 Event(function_call=...)
8. Runner 自动执行 get_weather("北京")，获取结果
   → Runner 产出 Event(function_response=...)
9. Runner 将工具结果回传模型，模型基于结果生成自然语言回复
   → Runner 逐 chunk 产出 Event(partial=True, text=...)
10. Runner 将完整的对话（用户消息 + 助手回复）保存到 Session
11. 下次对话时，Session 中已包含上轮历史 → 实现多轮对话
```

## 七、注意事项与踩坑

### 1. 环境变量必须在导入 Agent 之前加载

```python
load_dotenv()  # 先加载 .env
from agent.agent import root_agent  # 再导入 Agent（此时读取环境变量创建模型）
```

如果顺序反了，`get_model_config()` 读不到环境变量会抛 `ValueError`。

### 2. 同一个 session_id = 多轮对话

`Runner` 自动从 `SessionService` 按 `session_id` 加载消息历史。**只要 session_id 不变，就是多轮对话**。每次 `/clear` 命令生成新的 UUID 即可开启全新会话。

### 3. 流式 vs 非流式事件的区分

- `event.partial = True` → 流式文本 chunk，应该 `print(end="")` 拼接输出
- `event.partial = False` → 完整事件（工具调用 / 工具结果），应该单独一行展示

### 4. FunctionTool 依赖类型标注和 docstring

框架通过 **inspect** 提取函数签名和文档字符串来生成 Tool Schema（JSON Schema）。所以工具函数**必须**有：
- 参数类型标注（`city: str`）
- 返回类型标注（`-> dict`）
- docstring 描述功能
