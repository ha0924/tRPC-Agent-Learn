## 二、开源经历

### 2.1 本次自己参与的 Issue 链接和心得体会

> 当前正在系统性阅读 `trpc-group/trpc-agent-python` 仓库（v1.1.10）的核心模块源码，已围绕本次实战课题"多租户与节点部署"梳理出以下方向，准备提交 issue 与 PR：

1. **多租户原子能力补齐**：当前框架在 Session / Memory / Tool 层未原生提供 `tenant_id` 维度抽象，计划提交设计提案，将 tenant 上下文以 `contextvar` 方式贯穿 Runner 执行链路。
2. **Session 多节点并发一致性**：当前 Redis / SQL 实现下，多 Worker 并发写同一 session 的 event 顺序未严格保序，计划基于事件序号 + 乐观锁补一版强一致方案。
3. **IM Channel Adapter 扩展**：trpc-claw 已支持企业微信和 Telegram，计划补充微信客服与公众号 Adapter，并抽象出统一的 `IMChannel` 接口。

**心得体会**：阅读源码的过程让我深刻感受到 tRPC-Agent-Python 在"分层解耦 + 事件驱动"上的设计取舍——特别是 Runner 输出统一事件流、各能力服务（Session / Memory / Tool / Knowledge）通过统一接口注入的设计，与我在 360 工作流平台中接触到的"插件化能力总线"思路高度一致。这让我相信自己能够快速融入项目的设计语境，提出有价值的贡献。
